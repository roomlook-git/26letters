# 产品需求文档（PRD）：英语字母发音学习网页

**文档版本**：1.0  
**最后更新**：2026年1月29日  
**目标用户**：英语初学者、儿童、语言教师、自学用户  
**平台要求**：移动端优先（响应式设计，兼容主流移动浏览器）

---

## 1. 产品概述

开发一个轻量级、离线可用的网页应用，用于学习 **26个英文字母的两种核心发音**：
- **字母名称音（Letter Name）**：如 A → /eɪ/
- **字母自然音（Letter Sound）**：如 A → /æ/

未来可扩展至国际音标（IPA）、元音/辅音分类等内容，具备良好的架构延展性。

---

## 2. 核心功能需求

### 2.1 字母发音展示
- 展示全部 26 个英文字母（A–Z），每个字母独立区块。
- 每个字母区块需同时展示两项信息（仅展示，不单独播放）：
  - **字母名称音标（IPA）**：如 A → /eɪ/
  - **字母自然音标（IPA）**：如 A → /æ/
- 交互方式改为：**用户点击该字母区块（或其“播放”按钮）即可播放该字母的整段音频**，音频内已包含“字母名称音 + 自然音”两段内容（同一音频文件）。
- 交互控件需有明确可点击区域（≥48px），并提供播放状态反馈（例如高亮/正在播放标识）。

### 2.2 音频播放
- 音频应清晰、标准（建议使用美式发音）。
- 支持点击即播，自动停止当前正在播放的其他音频。
- 单个字母对应 **1 个音频文件**，音频内容顺序为：先“字母名称音”，后“自然音”（顺序固定，避免用户混淆）。
- 首次加载时可预加载关键音频以提升体验（但避免过大体积影响加载速度）。

### 2.3 移动端适配
- 页面布局为垂直滚动列表，单列展示字母。
- 点击区域足够大（≥48px），便于手指点击。
- 字体大小适中，确保在小屏设备上可读。
- 支持横屏/竖屏自适应。

---

## 3. 技术架构

### 3.1 前端
- **框架**：纯 HTML + CSS + JavaScript（无重型框架，保证轻量与离线可用）
- **响应式设计**：使用 Flexbox 或 Grid + viewport meta 标签
- **音频格式**：MP3（兼容性最佳）
- **交互逻辑**：原生 JavaScript 控制 `<audio>` 元素或 Web Audio API

### 3.2 数据存储（不使用数据库，推荐静态数据）
- 本项目不依赖数据库，使用 **静态数据文件（JSON）或直接写入 JS 常量** 管理 26 个字母的展示信息：
  - `symbol`：字母（A-Z）
  - `ipa_name`：字母名称音标（如 `/eɪ/`）
  - `ipa_sound`：字母自然音标（如 `/æ/`）
  - `audio_file`：对应音频文件路径（如 `audio/A.mp3`）
- 未来扩展到 IPA/更多内容时，可在同一数据结构中新增字段或新增分组数据文件，不影响现有播放逻辑。

### 3.3 音频资源管理
- 所有音频文件存放于 `/audio/` 目录。
- 命名规范：`{symbol}.mp3`  
  示例：`A.mp3`, `B.mp3`...
- 每个音频文件包含两段内容：字母名称音 + 自然音。
- 总体积控制在 **5–10MB 以内**，确保移动端快速加载。

### 3.4 API 设计（预留扩展）
- 虽当前为静态页面，但数据结构需支持未来通过 API 动态加载内容。
- 前端通过统一函数 `loadLetterData()` 获取字母发音项，底层可切换为：
  - 本地静态 JSON
  - 远程 REST API（如 `/api/phonemes?group=letters`）
- 此设计便于后续增加：
  - 48个国际音标（IPA）
  - 长元音 vs 短元音对比
  - 辅音及辅音组合（如 th, sh, ch）
  - 单词拼读练习模块

## 4. 非功能性需求

| 类别 | 要求 |
|------|------|
| **性能** | 首屏加载 < 2s（3G网络下），音频点击延迟 < 300ms |
| **离线支持** | 支持通过 Service Worker 缓存资源，实现离线使用 |
| **兼容性** | 支持 iOS Safari、Android Chrome 最新2个版本 |
| **可维护性** | 模块化代码结构，数据与视图分离 |
| **可扩展性** | 新增发音类型无需修改核心播放逻辑 |

### 4.1 部署与发布（无后端、纯静态）
- 站点以静态资源形式发布（HTML/CSS/JS/音频/Service Worker），不依赖数据库与服务端接口。
- 推荐免费部署方式：
  - **Cloudflare Pages**：自动 HTTPS、全球 CDN、支持自定义域名、与 GitHub 仓库联动发布。
  - **GitHub Pages**：简单稳定，适合纯静态站点与学习项目发布。
  - **Gitee Pages**：适合国内访问场景（具体免费额度与规则以 Gitee 现行政策为准）。

---

## 5. 未来扩展方向（Phase 2+）

1. **国际音标（IPA）模块**：展示48个音标，每个音标带发音、例词、口型图。
2. **对比练习**：如 /iː/ vs /ɪ/（sheep vs ship），支持双按钮对比播放。
3. **录音跟读**：用户录音并对比标准发音（需麦克风权限，后期引入）。
4. **进度跟踪**：记录用户已练习的字母/音标（使用 localStorage 或 IndexedDB）。
5. **多语言界面**：支持中文/英文界面切换（因用户偏好中文环境）。

## 6. 交付物

- 完整可运行的 HTML/CSS/JS 文件包
- `/audio/` 目录含 26 个音频文件（26 字母 × 1 文件；单文件内包含“字母名称音 + 自然音”）
- README.md：说明如何添加新发音内容

---

> ✅ **备注**：本方案避免依赖第三方语音合成 API（如 Web Speech API），因其在移动端兼容性差、无法精准控制音标发音，且不符合“离线可用”目标。采用预录专业音频是更可靠的选择。
